如何在个人博客上发布一篇文章

无标签
发布日期:   2025-05-25

本文将详细讲解如何在基于 Hexo 框架、使用闪烁之狐（Flicker Fox）主题的个人博客上发布第一篇文章，涵盖从文章创建、内容编辑、主题适配配置到本地预览、线上部署的全流程。

一、前期准备

在发布文章前，需确保你的 Hexo 博客已完成基础搭建和部署，核心准备工作如下：

  1. 环境就绪：本地已安装 Node.js（验证命令：node -v）和 Git（验证命令：git --version），且 Git 已配置好 GitHub 用户名和邮箱（配置命令：git config --global user.name "你的用户名"、git config --global user.email "你的邮箱"）。
  2. 博客部署完成：已初始化 Hexo 项目，成功安装闪烁之狐主题，并完成部署到 GitHub Pages（或其他平台），能通过线上链接访问博客首页。
  3. 编辑工具准备：使用 Typora、Notepad++ 等编辑器。

二、核心步骤：发布文章全流程

步骤 1：创建文章文件

Hexo 提供了便捷的命令行工具，可快速生成符合规范的文章文件，无需手动新建：

  1. 打开电脑终端（Windows：CMD 或 PowerShell；Mac：终端），进入你的 Hexo 项目根目录（即包含 _config.yml、themes 文件夹的目录），命令示例： cd /Users/xxx/Documents/blog（将路径替换为你的实际项目路径）。

  2. 执行创建文章的命令： hexo new "你的文章标题"（标题用英文引号包裹，可自定义，比如 hexo new "我的第一篇 Hexo 博客"）。

  3. 执行成功后，终端会提示类似 Created: ~/blog/source/_posts/我的第一篇 Hexo 博客.md 的信息，说明文章文件已生成。文件默认存储在 source/_posts/ 目录下，格式为 Markdown（.md），这是 Hexo 博客的标准文章格式。

步骤 2：编辑文章内容

生成的 .md 文件包含两部分核心内容：Front-matter 头部配置（主题核心适配项）和正文内容（Markdown 格式），需按以下要求编辑：

关键配置说明：

  • cover（封面图）：闪烁之狐主题主打封面展示，必须配置！需先在 Hexo 项目根目录的 source/ 文件夹下新建 images 文件夹，将准备好的封面图放入其中（比如 cover.jpg），路径直接写 /images/封面图文件名.格式 即可。建议封面图尺寸为 1200×600px，适配主题响应式布局，避免拉伸变形。

  • top: true：置顶功能需主题支持，闪烁之狐默认兼容，设置后文章会在首页顶部显示，适合重要内容。

步骤 3：本地预览文章

编辑完成后，建议先在本地预览文章效果，确认主题样式、排版、图片显示正常后再部署到线上：

  1. 保持终端在 Hexo 项目根目录，执行本地预览命令：

  2. 命令说明：

    • hexo clean：清理本地缓存文件（避免旧内容干扰）；
    • hexo g（全称 hexo generate）：生成静态文件（将 .md 文章转换为浏览器可识别的 HTML 文件）；
    • hexo s（全称 hexo server）：启动本地服务器。

  3. 命令执行成功后，打开浏览器访问 http://localhost:4000，即可看到博客首页。

步骤 4：部署文章到线上博客

本地预览无误后，即可将文章部署到线上（以部署到 GitHub Pages 为例，其他平台逻辑类似）：

  1. 确保终端仍在 Hexo 项目根目录，执行部署命令：

  2. 命令说明：hexo d（全称 hexo deploy）：将生成的静态文件推送到 GitHub 仓库（部署核心命令）。

  3. 首次部署可能会弹出 GitHub 登录验证窗口，按提示输入 GitHub 用户名和密码（或使用 Token 验证），完成授权即可。后续部署会自动记住权限，无需重复验证。

  4. 等待部署完成：终端显示 Deploy done: git 即表示部署成功。

步骤 5：验证线上发布效果

  1. 打开浏览器，访问你的线上博客地址（格式：https://你的GitHub用户名.github.io）。
  2. 若首页能看到刚发布的文章，点击进入详情页确认所有内容正常显示，说明发布成功。
  3. 若暂时未显示，可能是 GitHub Pages 缓存，等待 1-2 分钟后刷新即可。

三、常见问题解决

  1. 本地预览看不到文章？ - 检查文章文件是否在 source/_posts/ 目录下（路径错误会导致 Hexo 无法识别）； - 检查 Front-matter 格式是否正确（比如 : 后是否加空格、是否用 Tab 缩进），格式错误会导致文章无法解析。
  2. 线上博客看不到文章？ - 检查部署命令是否执行成功（终端是否显示 Deploy done: git）； - 登录 GitHub，进入你的博客仓库，查看 main 分支（或部署分支）是否有新的提交记录，确认文章对应的 HTML 文件已推送； - 检查 GitHub Pages 配置：仓库 → Settings → Pages → Source 是否选择了部署分支（比如 main）和根目录。
  3. 封面图不显示？ - 检查封面图路径是否正确（是否放在 source/images/ 目录下，路径是否写对）； - 检查图片文件名是否有特殊字符（建议用英文/数字命名，避免中文或空格）。
  4. 评论区不显示？ - 闪烁之狐主题的评论功能需手动配置，需在主题配置文件（themes/flicker-fox/_config.yml）中设置评论插件（比如 Valine、Gitee 评论），具体配置参考闪烁之狐主题官方文档。
